# heading tags
In HTML, heading tags range from `<h1>` to `<h6>`.
Their size decreases from **largest (`h1`)** to **smallest (`h6`)** by default (browser default styling).

Here are their approximate default sizes:

| Tag    | Default Size     | Description               |
| ------ | ---------------- | ------------------------- |
| `<h1>` | 32px (2em)       | Largest heading           |
| `<h2>` | 24px (1.5em)     | Slightly smaller          |
| `<h3>` | 18.72px (1.17em) | Medium heading            |
| `<h4>` | 16px (1em)       | Normal text size but bold |
| `<h5>` | 13.28px (0.83em) | Smaller heading           |
| `<h6>` | 10.72px (0.67em) | Smallest heading          |

---

### Important:

* These are **browser default sizes**.
* `1em = 16px` (default browser base font size).
* You can override sizes using CSS.

---

### Example:

```html
<h1>This is H1</h1>
<h2>This is H2</h2>
<h3>This is H3</h3>
<h4>This is H4</h4>
<h5>This is H5</h5>
<h6>This is H6</h6>
```

---

### If You Want Custom Sizes:

```html
<style>
h1 { font-size: 40px; }
h2 { font-size: 32px; }
h3 { font-size: 26px; }
h4 { font-size: 20px; }
h5 { font-size: 16px; }
h6 { font-size: 14px; }
</style>
```

### `<pre>` Tag in HTML

The `<pre>` tag stands for **preformatted text**.

It displays text **exactly as written in the HTML file**, preserving:

* âœ… Spaces
* âœ… Line breaks
* âœ… Tabs
* âœ… Formatting

Unlike normal HTML text, it does **not ignore extra spaces or new lines**.

---

## ðŸ”¹ Why Do We Use `<pre>`?

Normally in HTML:

```html
<p>Hello      World</p>
```

Output:

```
Hello World
```

Multiple spaces are ignored.

But with `<pre>`:

```html
<pre>
Hello      World
</pre>
```

Output:

```
Hello      World
```

Spacing is preserved.

---

## ðŸ”¹ Syntax

```html
<pre>
Your formatted text here
</pre>
```

---

## ðŸ”¹ Default Behavior

* Uses **monospace font** (like Courier)
* Preserves whitespace
* Maintains line breaks
* Displays text exactly as typed

---

## ðŸ”¹ Common Uses

1. Displaying **code snippets**
2. Showing **ASCII art**
3. Writing formatted poems
4. Showing structured text

---

## ðŸ”¹ Example (Code Display)

```html
<pre>
function greet() {
    console.log("Hello World");
}
</pre>
```

Output keeps indentation exactly.

---

## ðŸ”¹ Important Difference

| `<p>`                | `<pre>`               |
| -------------------- | --------------------- |
| Ignores extra spaces | Preserves spaces      |
| Ignores line breaks  | Preserves line breaks |
| Normal font          | Monospace font        |

---

## ðŸ”¹ Combining with `<code>`

Best practice for code:

```html

console.log(x);
</code></pre>
```

This keeps formatting + adds semantic meaning.
